# Quickstart

This guide will get you from zero to shipping code with Oh My OpenAgent in minutes.

<Note>
Make sure you've completed the [installation](/installation) and authenticated at least one model provider before continuing.
</Note>

## Your First Command: Ultrawork

The fastest way to experience Oh My OpenAgent is with a single command:

```bash
ultrawork
```

Or use the shorthand:

```bash
ulw
```

That's it. You don't need to know anything else.

### What Happens When You Type Ultrawork

When you include `ultrawork` or `ulw` in your prompt, the agent:

<Steps>
  <Step title="Analyzes Your Intent">
    The **Intent Gate** classifies what you actually want—not just the literal words you typed.
  </Step>

  <Step title="Explores the Codebase">
    **Sisyphus** fires up background agents to understand your project structure, conventions, and patterns.
  </Step>

  <Step title="Delegates to Specialists">
    Based on the task type, Sisyphus delegates to the right specialist:
    - **Hephaestus** for deep coding work
    - **Oracle** for architecture decisions
    - **Librarian** for documentation search
    - **Explore** for codebase grep
  </Step>

  <Step title="Implements in Parallel">
    Multiple agents work simultaneously—research, implementation, and verification happening in parallel.
  </Step>

  <Step title="Keeps Working Until Done">
    The **Todo Enforcer** ensures the agent doesn't go idle. The **Ralph Loop** keeps iterating until 100% complete.
  </Step>
</Steps>

### Example: Adding a Feature

Let's say you want to add user authentication to your app:

```text
ultrawork: Add JWT-based authentication with login and signup endpoints
```

The agent will:
1. Research your current API structure
2. Check for existing auth libraries
3. Implement the JWT logic
4. Add login/signup routes
5. Write tests
6. Verify with LSP diagnostics
7. Report completion

**You don't write a single line.** Just describe what you want.

## Using Prometheus for Planning

For more complex tasks, you want a plan before execution. That's where **Prometheus** comes in.

<Steps>
  <Step title="Enter Prometheus Mode">
    Press **Tab** in your OpenCode session to switch to Prometheus mode.

    Or explicitly invoke it:
    ```text
    @plan "Refactor authentication to use OAuth2"
    ```
  </Step>

  <Step title="Answer Interview Questions">
    Prometheus will interview you like a real engineer:
    - What OAuth providers do you want to support?
    - Should we keep JWT as a fallback?
    - Do you have existing user data to migrate?
    - What's the priority: speed or security features?
  </Step>

  <Step title="Review the Plan">
    Prometheus generates a detailed plan with:
    - Scope and assumptions
    - Task breakdown
    - Dependencies and order
    - Verification criteria
  </Step>

  <Step title="Execute with Atlas">
    Once you approve the plan, run:
    ```text
    /start-work
    ```

    **Atlas** takes over and orchestrates execution across specialized subagents.
  </Step>
</Steps>

<Note>
**When to use Prometheus:**
- Multi-day projects
- Critical production changes
- Complex refactoring
- When you need a documented decision trail
- When ambiguity exists in requirements
</Note>

## Understanding the Agents

### Sisyphus: Your Main Orchestrator

**Sisyphus** is the agent you'll interact with most. He's named after the Greek myth—rolls the boulder every day, never stops, never gives up.

```text
# Let Sisyphus handle everything
ultrawork: Fix all TypeScript errors in src/
```

Sisyphus will:
- Run diagnostics to find all errors
- Prioritize by severity
- Fix errors in parallel
- Verify with LSP
- Keep going until error count is zero

<Warning>
**Important:** Sisyphus works best with **Claude Opus 4.6**. Other models will work but may result in degraded performance.
</Warning>

### Hephaestus: The Deep Worker

**Hephaestus** runs on GPT-5.3 Codex and handles autonomous deep work. You give him goals, not recipes.

```text
# Explicitly delegate to Hephaestus
@hephaestus Implement a caching layer for our API
```

Hephaestus will:
- Research caching strategies
- Evaluate Redis vs in-memory options
- Implement the solution
- Add tests and documentation
- Verify performance

Use Hephaestus for:
- Complex architectural work
- Cross-domain knowledge synthesis
- Deep debugging across many files
- Research-heavy tasks

### Oracle: The Consultant

**Oracle** is your read-only high-IQ consultant. Use him for architecture decisions and complex debugging.

```text
@oracle Should we use GraphQL or REST for our new microservice?
```

Oracle will:
- Analyze your current architecture
- Consider team expertise and tooling
- Evaluate tradeoffs
- Provide a recommendation with reasoning

**Oracle doesn't write code.** He advises. Then you execute (or let Sisyphus execute).

### Librarian: Documentation Search

**Librarian** searches docs and OSS code to stay current on library APIs and best practices.

```text
@librarian How do I use Prisma's middleware feature?
```

Librarian will:
- Search official Prisma docs
- Find relevant code examples
- Explain the API
- Show usage patterns

### Explore: Fast Codebase Grep

**Explore** uses speed-focused models for pattern discovery across your codebase.

```text
@explore Find all files that import our deprecated auth module
```

Explore will:
- Use AST-aware search
- Return matching files with context
- Highlight specific lines
- Do it fast (uses Grok Code Fast 1 or MiniMax)

## Special Commands

### Ralph Loop

The **Ralph Loop** is a self-referential loop that doesn't stop until 100% done.

```text
/ulw-loop: Optimize all database queries for performance
```

The agent will:
- Profile current performance
- Identify slow queries
- Optimize one by one
- Re-profile after each change
- Keep iterating until benchmarks are met
- Never stop until complete

### Init Deep

Generate hierarchical `AGENTS.md` files throughout your project for better context:

```text
/init-deep
```

This creates:
```
project/
├── AGENTS.md              ← project-wide context
├── src/
│   ├── AGENTS.md          ← src-specific context
│   └── components/
│       └── AGENTS.md      ← component-specific context
```

Agents automatically read relevant context. Zero manual management.

### Background Agents

Fire multiple agents in parallel:

```text
ultrawork: Research React 19 features [background]
ultrawork: Audit our dependencies for security issues [background]
ultrawork: Generate API documentation from JSDoc comments [background]
```

All three run simultaneously. Results arrive when ready.

Skills are specialized workflows with embedded MCP servers.

The agent will:
- Launch a browser
- Navigate through your app
- Fill forms and click buttons
- Capture screenshots

Advanced git operations:


The agent will:
- Analyze uncommitted changes
- Group by logical units
- Create separate commits
- Write meaningful commit messages

## Understanding Output

### Hash-Anchored Edits

When the agent reads your files, you'll see content like:

```python
11#VK| function hello() {
22#XJ|   return "world";
33#MB| }
```

The `#VK`, `#XJ`, `#MB` are content hashes. They ensure edits are applied to the correct lines even if the file changed.

This is why Oh My OpenAgent has a 10x higher success rate on edit operations.

### Background Task Updates

When background agents complete, you'll see notifications:

```text
[Background: Research React 19] ✓ Complete
Key findings:
- New server actions API
- Automatic memo optimization
- Suspense improvements
```
