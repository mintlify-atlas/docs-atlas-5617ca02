## Overview

Multimodal Looker is a specialized agent for analyzing media files that require interpretation beyond raw text. It examines PDFs, images, and diagrams to extract specific information or provide summaries.

**Philosophy**: Extract ONLY what was requested, save context tokens by processing media offload.

## Key Characteristics

- **Mode**: `subagent` (invoked by other agents)
- **Temperature**: `0.1`
- **Default Model**: `gpt-5.3-codex` (medium variant)
- **Cost**: CHEAP

## Core Competencies

- PDF text and structure extraction
- Image description and layout analysis
- Diagram interpretation and flow explanation
- Table and data extraction from documents
- UI element identification in screenshots
- Chart and graph analysis

## Model Fallback Chain

1. **gpt-5.4** (medium) - OpenAI, OpenCode
2. **kimi-k2.5** - OpenCode Go
3. **glm-4.6v** - Z.ai Coding Plan
4. **gpt-5-nano** - OpenAI, GitHub Copilot, OpenCode

## Tool Restrictions

**Extremely restricted agent - READ ONLY**:

- **Allowed**: `read` (for media files only)
- **Denied**: ALL other tools (write, edit, bash, grep, task, etc.)
- **Question Permission**: Typically not used (focused on extraction)

## How It Works

1. **Receives**: File path + goal describing what to extract
2. **Reads and analyzes**: File deeply with vision capabilities
3. **Returns**: ONLY the relevant extracted information
4. **Saves tokens**: Main agent never processes the raw file

## Use Cases

### For PDFs

- Extract text from specific sections
- Parse table data
- Identify document structure (headings, sections)
- Extract metadata (author, date, version)
- Summarize specific pages or chapters

### For Images

- Describe UI layouts and elements
- Identify text in images (OCR)
- Describe visual composition
- Analyze charts and graphs
- Extract design patterns

### For Diagrams

- Explain relationships and flows
- Describe architecture depicted
- Identify components and connections
- Extract workflow steps
- Interpret system designs

## When to Use Multimodal Looker

**Use when:**
- Media files cannot be read as plain text
- Extracting specific information or summaries from documents
- Describing visual content in images or diagrams
- Need analyzed/extracted data, not raw file contents
- Want to save context tokens in main agent

**Avoid when:**
- Source code or plain text files (use Read tool directly)
- Files that need editing afterward (need literal content from Read)
- Simple file reading where no interpretation needed
- Text-based configuration files (JSON, YAML, etc.)

## Response Rules

1. **Return extracted information directly** - no preamble
2. **If info not found** - state clearly what's missing
3. **Match the language** of the request
4. **Be thorough on the goal** - concise on everything else
5. **Output goes straight to main agent** for continued work

## Delegation Best Practices

### Good Delegation

```typescript
task(
  subagent_type="multimodal-looker",
  description="Extract form fields from mockup",
  prompt="List all form fields in this mockup. For each field, provide: label, type, required/optional, placeholder text, and any validation rules shown.",
  files=["./designs/signup-form.png"]
)
```

**Why good**: Specific extraction goal, clear output format

### Bad Delegation

```typescript
task(
  subagent_type="multimodal-looker",
  description="Look at this image",
  prompt="What do you see?",
  files=["./image.png"]
)
```

**Why bad**: Vague goal, no specific extraction target, will return too much irrelevant info

## Context Token Savings

Multimodal Looker saves tokens by:

1. **Main agent never sees raw file** - only the extracted info
2. **Targeted extraction** - only relevant data returned
3. **Offloaded processing** - vision analysis happens in subagent

Example savings:
- 10-page PDF: ~20K tokens → 500 token extraction
- High-res screenshot: ~3K tokens → 200 token description
- Complex diagram: ~2K tokens → 300 token explanation
