## Overview

The root configuration schema for Oh My OpenAgent uses Zod v4 validation with `safeParse()`. All fields are optional - omitted fields use plugin defaults from each module.

Config files are JSONC format (JSON with comments) and support multi-level merging:

```
Project (.opencode/oh-my-opencode.jsonc) → User (~/.config/opencode/oh-my-opencode.jsonc) → Defaults
```

## Schema Location

Defined in `src/config/schema/oh-my-opencode-config.ts`

## Config Merging Rules

1. **Deep merge** for: `agents`, `categories`, `claude_code`
2. **Set union** (concatenate + dedupe) for: `disabled_*` arrays
3. **Override replaces** for all other fields
4. **Zod safeParse** fills defaults for omitted fields
5. **migrateConfigFile** transforms legacy keys automatically

## Schema Validation

All config files are validated with:

```typescript
import { OhMyOpenCodeConfigSchema } from "oh-my-opencode"

const result = OhMyOpenCodeConfigSchema.safeParse(configData)
if (!result.success) {
  console.error(result.error)
}
```
